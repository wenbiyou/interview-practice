# 🚀 部署指南

## 方案一：自动部署到 GitHub Pages（推荐）

### 步骤 1：创建 GitHub 仓库

1. 登录 GitHub：https://github.com
2. 点击右上角 **+** → **New repository**
3. 仓库名称填写：`interview-practice`（也可以是其他名称）
4. 选择 **Public**（公开仓库才能免费使用 GitHub Pages）
5. 点击 **Create repository**

### 步骤 2：上传代码

在本地项目目录执行：

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit"

# 添加远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/interview-practice.git

# 推送到 main 分支
git branch -M main
git push -u origin main
```

### 步骤 3：配置 GitHub Pages

1. 打开仓库页面
2. 点击 **Settings**（设置）
3. 左侧菜单点击 **Pages**
4. **Build and deployment** 部分：
   - **Source** 选择 **GitHub Actions**
5. 等待几分钟，GitHub Actions 会自动部署

### 步骤 4：修改配置文件

如果你的仓库名不是 `interview-practice`，需要修改 `vite.config.js`：

```javascript
export default defineConfig({
  plugins: [react()],
  base: '/你的仓库名/',  // 修改这里
  build: {
    outDir: 'dist'
  }
})
```

同时修改 `src/main.jsx`：

```javascript
<BrowserRouter basename="/你的仓库名/">
```

修改后重新提交：

```bash
git add .
git commit -m "Update base path"
git push
```

### 步骤 5：访问网站

部署完成后，访问地址：

```
https://你的用户名.github.io/interview-practice/
```

## 方案二：手动部署

### 本地构建并部署

```bash
# 安装依赖
npm install

# 构建生产版本
npm run build

# 部署到 GitHub Pages
npm run deploy
```

> 注意：手动部署需要配置 `gh-pages` 分支

## 方案三：部署到其他平台

### Vercel 部署

1. 访问 https://vercel.com
2. 点击 **Add New Project**
3. 导入 GitHub 仓库
4. 框架预设选择 **Vite**
5. 点击 **Deploy**

### Netlify 部署

1. 访问 https://www.netlify.com
2. 点击 **Add new site** → **Import an existing project**
3. 选择 GitHub 仓库
4. 构建设置：
   - Build command: `npm run build`
   - Publish directory: `dist`
5. 点击 **Deploy site**

## 📱 添加更多题目

### 编辑题目文件

打开 `src/data/questions.json`，按照以下格式添加题目：

```json
{
  "categories": [
    {
      "id": "html-css",
      "name": "HTML/CSS基础",
      "icon": "🎨",
      "tags": ["HTML", "CSS"],
      "questions": [
        {
          "id": 1,
          "title": "题目标题",
          "answer": "题目答案，支持<strong>HTML</strong>标签"
        }
      ]
    }
  ]
}
```

### 支持 HTML 标签

答案中支持以下 HTML 标签：
- `<h3>` - 三级标题
- `<p>` - 段落
- `<ul>` / `<li>` - 无序列表
- `<ol>` / `<li>` - 有序列表
- `<code>` - 行内代码
- `<pre>` / `<code>` - 代码块
- `<table>` / `<tr>` / `<th>` / `<td>` - 表格
- `<strong>` / `<b>` - 加粗

### 添加题目技巧

1. **批量转换**：可以将 Markdown 格式的答案转换为 HTML
2. **代码高亮**：使用 `<pre><code>` 包裹代码块
3. **题目分类**：建议按知识点分类整理
4. **ID 规则**：不同分类使用不同区间，如 HTML/CSS 用 1-100，JavaScript 用 101-200

## 🔧 自定义配置

### 修改主题色

编辑 `tailwind.config.js`：

```javascript
colors: {
  primary: {
    50: '#eff6ff',
    100: '#dbeafe',
    500: '#3b82f6',  // 主色
    600: '#2563eb',
    700: '#1d4ed8',
  }
}
```

### 修改网站标题

编辑 `index.html`：

```html
<title>你的刷题网站名称</title>
```

### 添加统计代码

在 `index.html` 的 `<head>` 中添加：

```html
<!-- Google Analytics -->
<script async src="https://www.googletagmanager.com/gtag/js?id=GA_MEASUREMENT_ID"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'GA_MEASUREMENT_ID');
</script>
```

## ❓ 常见问题

### 1. 部署后页面空白？

检查 `vite.config.js` 中的 `base` 是否与仓库名一致。

### 2. 刷新页面 404？

这是 GitHub Pages 单页应用的常见问题，需要配置 `_redirects` 或使用 hash 路由（本项目已使用 hash 路由）。

### 3. 如何更新网站？

修改代码后提交到 GitHub，Actions 会自动重新部署：

```bash
git add .
git commit -m "更新内容"
git push
```

### 4. 自定义域名？

1. 在仓库的 `Settings` → `Pages` → `Custom domain` 中添加域名
2. 在你的域名解析商处添加 CNAME 记录指向 `你的用户名.github.io`
3. 在项目根目录创建 `CNAME` 文件，内容为你的域名
