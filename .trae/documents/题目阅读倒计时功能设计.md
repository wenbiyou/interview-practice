# 题目阅读倒计时功能设计方案

## 需求分析

用户在刷题时，希望根据题目和答案的难易程度和字符长度，计算出前端开发看完所需要的时间，并显示倒计时。

## 核心功能

1. **时间估算算法** - 根据内容长度和难易程度计算阅读时间
2. **倒计时 UI 组件** - 显示剩余时间
3. **计时控制** - 开始/暂停/跳过/重置
4. **时间显示** - 友好的时间格式（分:秒）

## 时间估算算法设计

### 基础假设
- 普通开发者中文300-400阅读速度：** 字/分钟**
- 代码阅读速度：**100-150 行/分钟**
- 思考理解时间：**额外 30%**

### 计算公式

```
基础阅读时间 = (纯文本字符数 / 350) + (代码行数 / 120)
总时间 = 基础阅读时间 * 1.3 (思考理解)
```

### 难易程度系数
- 简单题：×0.7
- 中等题：×1.0
- 困难题：×1.5

### 预估规则（简化方案）

| 字符数范围 | 预估时间 |
|-----------|---------|
| 0-500 字  | 1 分钟  |
| 500-1000 字 | 2 分钟 |
| 1000-1500 字 | 3 分钟 |
| 1500-2500 字 | 4 分钟 |
| 2500-4000 字 | 5 分钟 |
| 4000+ 字 | 6 分钟 |

## UI 设计

### 倒计时组件位置
- 显示在题目卡片的顶部区域
- 在"查看答案"按钮下方显示

### 倒计时 UI 样式
- 圆形进度条或条形进度
- 剩余时间数字显示（大字体）
- 颜色变化：
  - 绿色：时间充裕 (>50%)
  - 黄色：时间紧张 (20%-50%)
  - 红色：时间不足 (<20%)

### 控制按钮
- ⏸️ 暂停/继续
- 🔄 重置
- ⏭️ 跳过（进入下一题）

## 预期效果

用户进入题目后：
1. 自动开始倒计时（显示预估阅读时间）
2. 用户可以点击"查看答案"
3. 阅读过程中可以看到剩余时间
4. 可以暂停/继续/跳过
5. 时间到了有强烈的视觉提醒（但不强制跳转）
